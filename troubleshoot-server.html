If http://localhost:8000 is not loading, follow these steps:

Step 1: Check if Python is Installed

Open Command Prompt and run:

python --version

If you see an error: Python is not installed

Solution: Download Python from python.org
Make sure to check "Add Python to PATH" during installation

Step 2: Check if Server is Running

In Command Prompt, make sure you see:

Serving HTTP on 0.0.0.0 port 8000 (http://0.0.0.0:8000/) ...

If you don't see this: The server isn't running

Solution: Make sure you're in the right folder and run the command again

Step 3: Check Your Folder Location

Make sure you're in the correct folder. In Command Prompt, run:

cd "D:\kkr project" dir

You should see files like: login.html, script.js, etc.

Step 4: Try Different Ports

If port 8000 is busy, try a different port:

python -m http.server 8080

Then open: http://localhost:8080

Step 5: Alternative Methods

Method A: Using Node.js (if you have it)

npx serve . -p 8000

Method B: Using PHP (if you have it)

php -S localhost:8000

Method C: Using Live Server (VS Code extension)

If you have VS Code, install "Live Server" extension and right-click on login.html ‚Üí "Open with Live Server"

Step 6: Check Firewall/Antivirus

Sometimes firewall or antivirus blocks local servers:

  • Check Windows Firewall settings
  • Add Python to antivirus exceptions
  • Try disabling antivirus temporarily

Step 7: Test Different URLs

Try these URLs in your browser:

http://127.0.0.1:8000 http://localhost:8000 http://localhost:8080

Step 8: Quick Test Without OAuth

If you just want to test the login page without OAuth, you can:

1. Open login.html directly in browser 2. Use email/password login (it will work) 3. Google/Facebook buttons will show helpful messages
